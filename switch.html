Conflit d'adresses potentiel
VLAN 137 (DMZ): SW2 utilise 192.168.137.1, mais c'est normalement l'IP de pfSense (gateway DMZ).
Recommandation: Modifier l'IP de SW2 en 192.168.137.2 pour éviter le conflit ARP.

Logique d'Ensemble

SW1 côté Proxmox
Rôle principal
• Fournit les VLAN à Proxmox via trunks et ports access
• Route entre tous les VLAN (17, 37, 47, 57, 67, 77, 107, 207)
• Fournit DHCP pour VLAN 47, 57, 67
• Port Fa1/0/48 en access VLAN 17 pour Proxmox
SW2 côté pfSense
Rôle principal
• Transporte les VLAN vers pfSense via trunk 802.1Q
• IP de management dans chaque VLAN (17, 107, 137)
• Ne joue PAS le rôle de gateway (c'est pfSense)
• Connecté à pfSense via Fa1/0/2 (trunk)
